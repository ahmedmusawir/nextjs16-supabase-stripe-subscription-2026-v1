# Changelog — v0.2.2
**Date:** 2026-04-12
**Type:** Patch — Form & Navigation UX Fixes (found during manual testing of v0.2.1)

---

## Summary

Eight UX improvements across forms and global navigation components. No database changes. No new dependencies.

---

## Changes

### 1. Confirm Password field added to Add User form
**File:** `add-user/AddUserForm.tsx`
- Added a `confirmPassword` field with Zod `.refine()` cross-field validation
- Error displays inline if passwords don't match: *"Passwords do not match"*
- `confirmPassword` is stripped before calling `addUser()` server action — only `name`, `email`, `password`, `role` are sent

### 2. All form error messages now visibly red
**Files:** `AddUserForm.tsx`, `EditUserForm.tsx`
- All `<FormMessage>` components previously used `dark:text-red-300` (invisible in light mode)
- Updated to `text-red-500 dark:text-red-400` — clearly visible in both light and dark mode

### 3. Redirect to dashboard on success — verified
**Files:** `AddUserForm.tsx`, `EditUserForm.tsx`
- Both forms already call `router.push("/superadmin-portal")` on success — behaviour confirmed correct, no change needed

### 4. Dropdown menu background — fixed globally
**File:** `src/components/ui/dropdown-menu.tsx`
- `DropdownMenuContent` was using `bg-popover` (CSS variable not resolving to a solid color in this project)
- Fixed to explicit `bg-white dark:bg-slate-800` with `border-slate-200 dark:border-slate-600` and proper text colors
- Applies to **all** dropdowns in the project (avatar menu, theme toggler, any future dropdowns)

### 5. Dropdown menu items — hover effect added + cursor pointer
**File:** `src/components/ui/dropdown-menu.tsx`
- `DropdownMenuItem` was using `cursor-default` with `focus:bg-accent` (accent CSS var not resolving)
- Changed to `cursor-pointer` with explicit `hover:bg-slate-100 dark:hover:bg-slate-700 focus:bg-slate-100 dark:focus:bg-slate-700`
- Applies to all dropdown items globally — avatar menu, theme toggler

### 6. Theme toggle icon — hover effect removed
**File:** `src/components/global/ThemeToggler.tsx`
- The Sun/Moon icon button was darkening on hover, giving the impression it was a different interactive state
- Fixed by setting `hover:bg-slate-700` to match the button background exactly — no visible change on hover

### 7. Avatar dropdown trigger — finger pointer cursor on all navbars
**Files:** `Navbar.tsx`, `NavbarSuperadmin.tsx`, `NavbarHome.tsx`
- `DropdownMenuTrigger` wrapping the Avatar had no cursor class — defaulted to `cursor-default`
- Added `className="cursor-pointer"` to all three navbar avatar triggers

### 8. Avatar dropdown content background — fixed on all navbars
**Files:** `Navbar.tsx`, `NavbarSuperadmin.tsx`, `NavbarHome.tsx`
- `DropdownMenuContent` had inline `className="bg-white dark:bg-slate-600"` or no class at all in some navbars — inconsistent
- Normalised to `bg-white dark:bg-slate-800` across all three

## Notes

- `dropdown-menu.tsx` and `dialog.tsx` both had the same root issue: shadcn CSS variables (`bg-popover`, `bg-background`) were not resolving to solid colors because the project's Tailwind/CSS setup doesn't define them. Always use explicit Tailwind color classes (`bg-white dark:bg-slate-800`) for any new shadcn components added to this project.
- The confirm password field is client-side only — it is validated by Zod and never sent to the server action.
