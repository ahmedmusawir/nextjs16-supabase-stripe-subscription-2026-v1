## Summary

Second QA pass: eliminated the fullscreen loading overlay that hid the Navbar, fixed superadmin data leak in admin portal, added pagination spinner with `useTransition`, and improved the registration form UX.

---

## Bug Fixes

### 1. Disappearing Navbar — NavigationLoadingProvider Removed
**Files:** `src/app/layout.tsx`, `src/components/providers/NavigationLoadingProvider.tsx` (DELETED)

- **Root cause:** `NavigationLoadingProvider` intercepted ALL `<a>` clicks and rendered a `fixed inset-0 z-[9999] bg-white` fullscreen overlay. This covered the entire page — Navbar, sidebar, everything — during every route transition.
- **Fix:** Deleted `NavigationLoadingProvider.tsx` entirely and removed it from root `layout.tsx`. Next.js `loading.tsx` files now handle transitions correctly.
- **Additionally:** Moved `loading.tsx` files from route group level into portal subdirectories so the spinner only replaces page content (Navbar + sidebar stay visible):
  - `(superadmin)/loading.tsx` → `(superadmin)/superadmin-portal/loading.tsx`
  - `(admin)/loading.tsx` → `(admin)/admin-portal/loading.tsx`
  - `(members)/loading.tsx` → `(members)/members-portal/loading.tsx`

### 2. Superadmin Leak in Admin Portal
**File:** `src/app/(admin)/admin-portal/actions.ts`

- **Bug:** Admin portal's `getUsers` queried ALL profiles from the database. The component-level `.filter()` was removed in v0.4.0 but the DB-level fix was only applied to the superadmin portal's `getUsers`.
- **Fix:** Applied the same 3-query pattern: pre-query `user_roles.neq("role", "superadmin")` to get allowed IDs, then filter `profiles.in("id", allowedIds)` for accurate count and pagination.

### 3. Pagination Spinner (useTransition)
**Files:** `src/components/common/PaginationControls.tsx` (CREATED), `SuperadminPortalPageContent.tsx`, `AdminPortalPageContent.tsx`

- **Bug:** Clicking "Next" or "Previous" gave no visual feedback — the page appeared frozen until the server responded.
- **Fix:** Created a shared `PaginationControls` client component that uses `useRouter()` + `useTransition()`. When `isPending`, a `Loader2` spinner appears next to the page indicator and buttons are disabled. Both portals now use this component.

### 4. Registration Error Text Color
**File:** `src/components/auth/RegisterForm.tsx`

- **Bug:** `FormMessage` components had `className="dark:text-red-300"` but no light-mode red — errors rendered in black on light backgrounds.
- **Fix:** Added `text-red-500` to all 4 `FormMessage` elements.

### 5. Registration Button Loading Spinner
**File:** `src/components/auth/RegisterForm.tsx`

- **Bug:** Clicking "Signup" triggered the auth request with no visual feedback on the button.
- **Fix:** Added `isLoading` state. Button is disabled during the request and shows `Loader2 animate-spin` with "Signing up..." text.
