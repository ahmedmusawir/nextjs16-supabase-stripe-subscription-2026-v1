## Bug Fixes

### 1. Loading/Navbar Viewport Bug (members portal)
**File:** `src/app/(members)/loading.tsx`

- **Bug:** `min-h-screen bg-white` on the loading state painted the entire viewport white, hiding the Navbar during page transitions.
- **Fix:** Changed to `min-h-[60vh]` with no background override — matches the pattern used by `(admin)` and `(superadmin)` loading states.

---

### 2. Public Signup — Name Not Saved to Database
**Files:** `src/app/api/auth/signup/route.ts`, `src/components/auth/RegisterForm.tsx`

- **Bug:** `supabase.auth.signUp()` was called with `options.data.name` but the DB trigger reads `full_name`. Name was silently dropped.
- **Fix:** Changed key to `full_name` so the trigger correctly populates the `profiles` table.
- **Also fixed 3 copy-paste errors in RegisterForm:**
  - Name field placeholder: `"Please Enter Email"` → `"Enter your full name"`
  - Confirm Password label: `"Password"` → `"Confirm Password"`
  - Confirm Password placeholder: `"Please Enter password"` → `"Confirm your password"`

---

### 3. Superadmin Role Dropdown Security Leak
**File:** `src/app/(superadmin)/superadmin-portal/edit/[id]/EditUserForm.tsx`

- **Bug:** Role dropdown included `"Superadmin"` as a selectable option — critical security risk allowing privilege escalation.
- **Fix:** Hardcoded dropdown to `["admin", "member"]` only. Removed `superadmin` from the Zod enum and the SelectItem list. Guard added: if existing user has `role: "superadmin"`, their form defaults to `"admin"`.

---

### 4. Pagination Math Bug (Superadmin + Admin Portals)
**Files:** `src/app/(superadmin)/superadmin-portal/actions.ts`, `src/app/(superadmin)/superadmin-portal/SuperadminPortalPageContent.tsx`, `src/app/(admin)/admin-portal/AdminPortalPageContent.tsx`

- **Bug:** `getUsers` returned `total` count from the raw `profiles` table (including superadmins). The component then filtered out superadmins in JS. So with 7 users (1 superadmin + 6 others), `totalPages = ceil(7/6) = 2`, but page 2 from the DB had 0 non-superadmin records → "No users found."
- **Fix:** `getUsers` now performs a pre-query on `user_roles` to get all non-superadmin user IDs, then queries `profiles` filtered to those IDs. The `count` returned is now accurate, and pagination math is correct. Component-level `filter()` removed from both portals.

---

### 5. Password Validation Errors — Toast Replaced with Inline Text
**Files:** `src/app/(members)/members-portal/profile/ProfileForm.tsx`, `src/app/(admin)/profile/ProfileForm.tsx`

- **Bug:** "Passwords do not match" validation triggered a bottom-left toast notification that overlapped the Submit button, making it unreadable.
- **Fix:** Removed `toast()` calls for client-side validation errors. Added a local `validationError` state string. Inline `<p className="text-sm text-red-500 dark:text-red-400 mt-1">` renders directly below the Confirm Password input.

---

### 6. Name Capitalization on Dashboards
**Files:** `src/app/(superadmin)/superadmin-portal/SuperadminPortalPageContent.tsx`, `src/app/(admin)/admin-portal/AdminPortalPageContent.tsx`

- **Bug:** User names appeared in all lowercase on dashboard cards if entered that way.
- **Fix:** Added Tailwind `capitalize` class to `<CardTitle>` name display in both portals. Names already entered in DB appear title-cased in the UI without requiring a data migration. New saves are capitalized at the server action level (see v0.3.x).

---

### 7. Login Error Bubbling to Next.js Dev Overlay
**File:** `src/components/auth/LoginForm.tsx`

- **Bug:** Wrong-password login threw an error that surfaced in the Next.js 15 / React 18 development error overlay before the component's `try/catch` could process it cleanly.
- **Fix:** Replaced `try/catch` with `.catch()` chain on the login promise. This intercepts the rejection at the Promise level before React's error boundary observes it. Error is still displayed via inline `setError()` state.

## New Features

### 8. Admin Portal — Add Member
**Routes:** `/admin-portal/add-member`
**Files:** `src/app/(admin)/admin-portal/add-member/page.tsx`, `AddMemberForm.tsx`, `actions.ts` (addMember appended)

- Admins can create new users with `role: "member"` only
- Form: Full Name, Email, Temporary Password, Confirm Password
- Name auto-capitalized via `toTitleCase()` before saving
- Duplicate email detection with user-friendly error message
- Redirects to dashboard on success

### 9. Admin Portal — Profile Page
**Route:** `/profile`
**Files:** `src/app/(admin)/profile/page.tsx`, `ProfileForm.tsx`

- Email displayed read-only, role badge in red (Admin)
- Password update form with inline validation errors

### 10. Member Portal — Profile Page
**Route:** `/members-portal/profile`
**Files:** `src/app/(members)/members-portal/profile/page.tsx`, `ProfileForm.tsx`

- Initials avatar (green circle, first 2 initials of full name)
- Name + email displayed read-only with note to contact admin for changes
- Role badge in green (Member)
- Password update form with inline validation errors
- Sidebar "Profile" link wired to `/members-portal/profile`

### 11. Name Auto-Capitalization (Server Actions)
**Files:** `src/app/(superadmin)/superadmin-portal/actions.ts`, `src/app/(admin)/admin-portal/actions.ts`

- `toTitleCase()` helper applied to `full_name` in `addUser`, `editUser`, `addMember`
- Converts any casing (`"john doe"`, `"TONY STARK"`) to `"John Doe"`, `"Tony Stark"` before writing to DB

### 12. Toast Background Fix
**File:** `src/components/ui/toast.tsx`

- Default toast variant changed from `bg-background` (transparent in some contexts) to `bg-white dark:bg-zinc-900` — always opaque.
