## Worked Examples (For Reference)

### Example 1: Dead Code

```markdown
### Dead priceAfterDiscount calculation in orderTransform.ts

- **Location:** `src/lib/orderTransform.ts` (lines 47-62)
- **Description:** A `priceAfterDiscount()` helper function exists with full logic for applying coupon codes to line items, but is never called anywhere. Discount logic now lives in `src/lib/checkout/applyCoupons.ts`. The helper was left over from the v0 refactor.
- **Fix path:** Delete the function. Run `npm test` to confirm no tests reference it. Remove its (also unused) unit tests in `tests/lib/orderTransform.test.ts`.
- **Risk if left:** ⚪ VERY LOW — Inert dead code, no security implication, just hygiene.
- **Notes:** Consider doing as part of the next general cleanup pass; don't make a dedicated sprint for one function.

---
```

### Example 2: Deprecated Pattern

```markdown
### Sync params usage in route handlers (Next 15 deprecation)

- **Location:** Repo-wide — affects `src/app/api/users/[id]/route.ts`, `src/app/products/[slug]/page.tsx`, and 4 other files
- **Description:** Next 15 introduced async `params` (`const { id } = await params` instead of `const { id } = params`). Currently using sync pattern, which works but emits deprecation warnings during build. Future Next versions may remove sync support entirely.
- **Fix path:** Sweep all `[param]` route handlers and pages, convert to async `params` pattern. Roughly 6 files. Done in a single PR.
- **Risk if left:** 🟡 MEDIUM — Will break on a future Next major version bump. Build warnings clutter the output now.
- **Notes:** Do before any major Next upgrade. Consider doing alongside Phase 2 (Visual Fidelity) since that phase touches several pages anyway.

---
```

### Example 3: Lint Warning Cluster

```markdown
### Unused imports across components

- **Location:** Repo-wide — ~18 components based on eslint output
- **Description:** Several components have unused import statements left over from refactors. ESLint flags them but they're not errors. Adds noise to the diff when re-reading files.
- **Fix path:** Run `npx eslint --fix src/` for auto-resolvable cases. Manually review the rest. Verify build still passes.
- **Risk if left:** 🟢 LOW — Cosmetic clutter, slightly inflated bundle (tree-shaking should eliminate, but cleaner code is cleaner code).
- **Notes:** Quick win — 15 minutes of focused work.

---
```

## A Note On Cleanup Discipline

Cleanup backlogs tend to grow forever. To keep this file useful:

1. **Don't capture every TODO.** This is for hygiene with structure — patterns, clusters, real debt. Not every "could be cleaner" thought.
2. **Move resolved items promptly.** A bloated `## Open Items` section with already-fixed entries becomes noise.
3. **Periodically prune.** If a VERY LOW item has been open for 12+ months and the file has been touched many times since, it's effectively WON'T FIX. Either commit to it or remove the entry with a note.
4. **Cluster related items.** Five separate "remove unused import in file X" entries should be ONE "Unused imports across components" entry.
